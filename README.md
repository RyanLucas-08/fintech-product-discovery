# 💳 Estudo de Caso: Onboarding & Segurança em Fintech

Este repositório documenta a fase de **Product Discovery** e **Análise Ágil** para o módulo de entrada de uma Fintech. O objetivo foi aplicar conceitos de Engenharia de Requisitos, Princípios INVEST e Matriz de Priorização para um MVP.

## 🎯 O Épico: Onboarding e Segurança
O foco deste estudo é garantir que a "porta de entrada" do usuário seja intuitiva, mas extremamente segura, estabelecendo confiança desde o primeiro contato.

## 🛠️ User Stories & Critérios de Aceitação
Neste projeto, estruturei 5 histórias de usuário utilizando a sintaxe **Gherkin** (Dado/Quando/Então):
1. **Interface Intuitiva:** Foco em UX e primeira impressão.
2. **Cadastro Simplificado:** Agilidade na coleta de dados essenciais.
3. **Armazenamento de Dados:** Persistência e integridade das informações.
4. **Autenticação Biométrica:** Segurança avançada com reconhecimento facial.
5. **Recuperação de Acesso:** Retenção de usuários e fluxo de segurança (Tokens).

## ⚖️ Critérios de Avaliação (Matriz de Decisão)
Para cada funcionalidade, realizei uma análise baseada em:
- **Valor ao Usuário:** O impacto real na dor do cliente.
- **Risco:** Análise de vulnerabilidades e perdas de negócio.
- **Dependências:** Mapeamento técnico (ex: Banco de Dados, APIs Externas).
- **Simplicidade:** Estimativa de esforço vs. complexidade técnica.
- **Impacto no MVP:** O que é vital para o lançamento da Versão 1.0.

## 🧠 Lições Aprendidas
- A importância de validar histórias com o **INVEST**.
- Como priorizar funcionalidades complexas (como Biometria) em um cenário de MVP.
- A visão de que o programador deve entender o fluxo do negócio para evitar retrabalho técnico.
